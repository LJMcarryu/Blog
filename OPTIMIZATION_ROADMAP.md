# 博客优化路线图：对标优秀个人博客

> 生成日期：2026-03-02
> 对标参考：antfu.me、leerob.io、joshwcomeau.com、overreacted.io
> 当前完成度：80%+（基础架构、SEO、i18n、暗色模式、代码高亮、评论系统均已完成）

---

## 第一优先级：体验差距明显的短板

### 1. 全文搜索（Pagefind）
**现状**：只能搜标题和描述，无法搜正文内容
**对标**：antfu.me 用 pagefind，leerob.io 用 Algolia

> 这是用户体验上最大的缺失。文章增多后，URL query 搜索几乎不可用。

### 2. 真实照片与项目内容
**现状**：照片是 SVG 占位、项目只有 1 个 demo、社交链接是 `#`、邮箱是 `your@email.com`
**对标**：所有优秀博客的个人品牌感都来自真实内容

> 技术再好，占位内容会让访客立刻离开。这是上线前必须替换的。

### 3. 文章数量
**现状**：4 篇（且都是建站相关）
**对标**：优秀博客通常 10+ 篇起步，涵盖不同主题

> 至少需要 8-10 篇有深度的原创文章才能让博客有"可逛性"。

## 第二优先级：从"能用"到"出色"

### 4. 阅读体验增强

| 缺失项 | 说明 | 对标 |
|--------|------|------|
| **文章系列/专栏** | 相关文章归组，如"React 系列" | joshwcomeau.com 的课程系列 |
| **文章内锚点链接** | h2/h3 hover 显示 `#` 链接图标，可复制分享 | antfu.me, MDN |
| **阅读进度条** | 页面顶部细条显示阅读进度 | 许多技术博客标配 |
| **图片 caption** | MDX 图片支持说明文字 | 技术文章常需要图注 |
| **脚注支持** | `remark-gfm` 支持脚注语法 | 学术/深度文章常用 |

### 5. 性能优化

| 缺失项 | 说明 | 对标 |
|--------|------|------|
| **字体优化** | 当前用系统字体栈，中文排版可考虑加载 Inter（英文）+ 指定中文字体 | antfu.me 用 Inter |
| **预加载/预获取** | 鼠标悬停链接时 prefetch 下一页 | Next.js Link 默认支持，但可验证是否生效 |
| **图片格式** | 上线后照片应使用 WebP/AVIF | leerob.io |
| **Bundle 分析** | 未见 `@next/bundle-analyzer`，不知道哪些依赖最重 | 生产标配 |

### 6. 交互细节

| 缺失项 | 说明 | 对标 |
|--------|------|------|
| **页面过渡动画** | 路由切换时无过渡，略显生硬 | antfu.me 的 View Transitions |
| **外部链接图标** | 外链旁无标识，用户不知道会跳走 | 常见 UX 模式 |
| **代码块文件名** | 代码块顶部不显示文件名/语言标签 | rehype-pretty-code 支持 `title` meta |
| **目录折叠** | TOC 当前始终展开，长文章 TOC 很长 | 可折叠或固定在侧边栏 |

## 第三优先级：锦上添花

### 7. 社交与互动

| 项目 | 说明 |
|------|------|
| **文章分享按钮** | Twitter/微博/复制链接一键分享 |
| **文章反应/点赞** | 类似 GitHub reactions 或简单计数（可用 Supabase） |
| **阅读量统计** | 显示每篇文章阅读次数（Umami 或自建） |
| **Newsletter 订阅** | 邮件列表入口（Buttondown/Resend） |

### 8. 开发者体验与工程化

| 项目 | 说明 |
|------|------|
| **E2E 测试** | Playwright 覆盖关键流程（目前只有单元测试） |
| **CI/CD pipeline** | GitHub Actions：lint + test + build 自动检查 |
| **Commit lint** | commitlint + husky 规范提交信息 |
| **内容校验** | MDX frontmatter schema 校验（zod），防止忘填 date/title |
| **Preview 部署** | Vercel Preview Deployments 给 PR 预览 |

### 9. 高级 SEO

| 项目 | 说明 |
|------|------|
| **canonical URL** | 中英文文章互指 `hreflang` alternate |
| **面包屑 JSON-LD** | 导航层级结构化数据 |
| **文章更新时间** | frontmatter 加 `updatedDate`，OG + JSON-LD 中体现 |

### 10. 视觉差异化

| 项目 | 对标参考 |
|------|----------|
| **自定义 MDX 组件** | 警告框、提示框、可交互 demo（joshwcomeau.com 的核心竞争力） |
| **手绘/手写风格元素** | 让博客有个人辨识度，不只是"又一个 dev blog" |
| **首页 hero 动画** | 简单的打字机效果或 greeting 动画 |

## 优先级矩阵

| 优先级 | 项目 | 难度 | 影响 |
|--------|------|------|------|
| **P0 上线前必做** | 替换占位内容（照片/项目/链接/邮箱） | 低 | 极高 |
| **P0 上线前必做** | 写更多文章（至少 8-10 篇） | 中 | 极高 |
| **P1 显著提升** | Pagefind 全文搜索 | 中 | 高 |
| **P1 显著提升** | heading 锚点链接 + 代码块文件名 | 低 | 高 |
| **P1 显著提升** | GitHub Actions CI | 低 | 高 |
| **P2 体验打磨** | 阅读进度条 | 低 | 中 |
| **P2 体验打磨** | 页面过渡动画（View Transitions） | 中 | 中 |
| **P2 体验打磨** | 外部链接图标 | 低 | 中 |
| **P2 体验打磨** | hreflang 跨语言 SEO | 低 | 中 |
| **P3 锦上添花** | 自定义 MDX 组件（Callout 等） | 中 | 中 |
| **P3 锦上添花** | 文章分享/点赞/阅读量 | 高 | 低 |
| **P3 锦上添花** | Newsletter 订阅 | 中 | 低 |

---

## 一句话总结

技术架构已经很完善，最大的差距不在代码，在**内容和真实感**。先填充真实内容，再做 Pagefind 搜索和阅读体验细节打磨，就是一个非常扎实的个人博客了。
