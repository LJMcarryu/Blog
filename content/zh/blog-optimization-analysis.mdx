## 为什么要做这个分析

博客搭完、功能也加了不少，但始终有个问题萦绕：**和真正优秀的个人博客比，我还差什么？**

于是我挑了几个公认做得好的开发者博客做对标分析：

- **antfu.me**（Anthony Fu）——极简设计、流畅动画、开发者工具作者
- **leerob.io**（Lee Robinson）——Vercel VP，Next.js 最佳实践标杆
- **joshwcomeau.com**（Josh W. Comeau）——交互式教程、自定义 MDX 组件
- **overreacted.io**（Dan Abramov）——极致简约、内容为王

## 当前完成度：80%+

经过 7 轮迭代，我的博客已经覆盖了大部分核心功能：

- ✅ Next.js 16 App Router + React Server Components
- ✅ 中英双语（next-intl）
- ✅ 暗色模式（零闪烁 + View Transitions）
- ✅ 代码高亮（Shiki 双主题）
- ✅ SEO 全套（sitemap、robots、RSS、OG 图片、JSON-LD）
- ✅ 标签系统、目录、分页
- ✅ 评论系统（Giscus）
- ✅ 单元测试（Vitest，17 项通过）

但对比顶级博客，**差距主要在三个层面**。

## 第一层差距：内容与真实感

这是最大的差距，而且**无法用代码解决**。

| 现状 | ��题 |
|------|------|
| 照片是 SVG 占位图 | 访客一眼看出是假的 |
| 项目只有 1 个 demo | 无法体现技术广度 |
| 社交链接是 `#` | 点了没反应，体验很差 |
| 文章只有 4 篇 | 太少，没有"可逛性" |

> antfu.me 的吸引力不在于技术实现多华丽，而在于**内容是真实的、持续更新的**。

### 行动项

- 替换所有占位内容（照片、项目、社交链接、邮箱）
- 至少写到 10 篇有深度的原创文章
- 覆盖不同主题，不只是"我怎么建博客"

## 第二层差距：阅读体验细节

这些是"有了会让用户多待一会"的细节打磨：

### 全文搜索

当前搜索只能匹配标题和描述，**正文内容完全搜不到**。文章增多后这个问题会越来越严重。

antfu.me 用的是 [Pagefind](https://pagefind.app/)——构建时生成索引，纯静态，无需后端。这是最适合静态博客的方案。

### Heading 锚点链接

技术文章经常需要分享某个章节的链接。优秀博客的 h2/h3 旁边都会有一个 `#` 图标，hover 可见，点击可复制。

```tsx
// 理想效果
<h2 id="why-nextjs">
  <a href="#why-nextjs" class="anchor">#</a>
  Why Next.js?
</h2>
```

### 代码块增强

当前代码块只有语法高亮和复制按钮。还缺少：

- **文件名/语言标签**：`rehype-pretty-code` 支持 `title` meta
- **行号显示**：长代码块需要行号辅助定位
- **diff 高亮**：展示代码变化时用绿/红标记

### 阅读进度条

页面顶部一条细线，跟随滚动显示阅读进度。实现简单，但对长文章的阅读体验提升明显。

### 页面过渡动画

当前路由切换是硬切，没有过渡效果。antfu.me 使用 View Transitions API 实现了丝滑的页面过渡。

## 第三层差距：工程化与高级 SEO

### CI/CD

目前没有 GitHub Actions。意味着：
- 推了有语法错误的代码不会被拦截
- 测试写了但没有自动运行
- PR 没有自动预览

一个基础的 CI pipeline 只需要：

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run lint
      - run: npm test
      - run: npm run build
```

### hreflang 跨语言 SEO

中英文文章是同一内容的不同语言版本，但搜索引擎不知道这一点。需要在 `<head>` 中添加 `hreflang` alternate 链接，告诉 Google "这两个 URL 是同一篇文章的中文/英文版"。

### 内容校验

MDX 的 frontmatter 没有 schema 校验。如果忘了写 `date` 或 `title`，要到运行时才发现。可以用 zod 在构建时做校验。

## 结论

对比下来发现一个规律：**技术实现上的差距其实不大**。我的博客在架构、SEO、i18n、暗色模式等方面已经达到了顶级博客的水准。

真正的差距在于：

1. **内容**——这需要时间和持续输出
2. **细节打磨**——锚点链接、进度条、搜索这些"最后 20%"
3. **持续运营**——博客不是搭完就结束，而是一个长期项目

技术只是载体，内容才是灵魂。接下来的重点应该是**写更多有价值的文章**，而不是继续折腾技术实现。
