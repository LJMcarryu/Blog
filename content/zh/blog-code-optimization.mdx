## 背景

[上一篇文章](/zh/blog/blog-code-review)做了一次全面的代码审查，列出了 15 个优化点。这篇文章记录实际的修复过程——最终远不止 15 项，7 轮迭代下来一共完成了 45+ 项改进。

---

## 第一轮：高优先级修复

### 环境变量集中管理

之前 `robots.ts`、`sitemap.ts`、`feed.xml/route.ts`、`layout.tsx`、`blog/[slug]/page.tsx` 五个文件各自写了一遍 `process.env.NEXT_PUBLIC_SITE_URL || "https://example.com"`。

新建 `lib/env.ts`，抽取 `getSiteUrl()` 函数，统一处理 fallback 并在缺失时输出 `console.warn`：

```typescript
export function getSiteUrl(): string {
  const url = process.env.NEXT_PUBLIC_SITE_URL;
  if (!url) {
    console.warn("[env] NEXT_PUBLIC_SITE_URL is not set — falling back to", FALLBACK_URL);
    return FALLBACK_URL;
  }
  return url;
}
```

### Lightbox 无障碍访问

原来的 Lightbox 只是一个 `div` 模拟的弹窗，键盘用户可以 Tab 跳出去。修复后：

- 添加 `aria-modal="true"`
- 打开时自动聚焦关闭按钮
- 锁定 `body` 滚动
- 实现焦点陷阱（Tab 循环在 lightbox 内部）

### CSS 清理

- 代码块背景色 `#fafafa` / `#161b22` → `var(--code-bg)`
- 代码文字色 `#0a0a0a` / `#f0f0f0` → `var(--fg-deeper)` / `var(--fg)`
- 给所有必要的 `!important` 加上注释说明原因

### 其他修复

- OG 图片 title 截断到 100 字符
- MDX 解析错误加 `console.warn` 日志
- 硬编码的"全部"/"上一篇"/"下一篇"移入 i18n messages
- RSS `language` 从 `"zh-en"` 修正为 `"zh"`
- 布局 metadata description 改为 SEO 友好的英文
- `CodeCopyButton` 的 `setTimeout` 替换为 `MutationObserver`
- `ThemedVideo` 改为双 video 元素 + CSS visibility 切换
- SKILLS 数据从 about 页面抽离到 `data/skills.ts`
- i18n 类型断言改用 `(typeof routing.locales)[number]`

## 第二轮：组件优化

### TableOfContents 同步修复

和 CodeCopyButton 一样，把 150ms 的 `setTimeout` 替换为 `MutationObserver`，确保 MDX 渲染完成后再初始化目录。

### 分页组件 i18n

`Pagination.tsx` 里的"上一页"/"下一页"也是硬编码的。改为通过 props 传入翻译后的标签，移除了 `locale` prop。

### Giscus 评论主题切换优化

之前每次切换主题都会销毁并重建整个 Giscus iframe。改用 Giscus 的 `postMessage` API 原地更新主题：

```typescript
iframe.contentWindow?.postMessage(
  { giscus: { setConfig: { theme } } },
  "https://giscus.app"
);
```

### 小修复

- 星级评分的 `#facc15` → `var(--star)` CSS 变量
- ThemedVideo 只对可见的视频设置 `autoPlay` 和 `preload="auto"`

## 第三轮：安全与 SEO

`next.config.ts` 从空配置升级为：

```typescript
const nextConfig: NextConfig = {
  poweredByHeader: false,

    return [{
      source: "/(.*)",

        { key: "X-Content-Type-Options", value: "nosniff" },
```

### SEO 元数据补全

## 第五轮：设计令牌体系化

这是最大的一次 CSS 重构。项目中大量使用 `rgba(125, 125, 125, 0.2)` 和 `rgba(125, 125, 125, 0.1)` 作为边框和背景色，散落在 CSS 和 JSX inline style 中。

新增两个设计令牌：

```css
:root {
  --border:     rgba(125, 125, 125, 0.2);
  --bg-subtle:  rgba(125, 125, 125, 0.1);
}
```

替换了 9 处 CSS 选择器和 7 处 JSX inline style，统一走变量体系。

同时修复了 `ThemeToggle.tsx` 中不必要的 `as unknown as` 双重类型断言——项目的 TypeScript DOM lib 已经支持 `startViewTransition`。

## 第六轮：深层清理与可访问性

第二轮审查后发现的问题，集中在 CSS 变量覆盖不彻底、可访问性遗漏和冗余代码三个方面。

### CSS 变量体系补全

`globals.css` 中仍有 15+ 处裸写的 `rgba(125, 125, 125, ...)` 值。新增 7 个设计令牌，彻底消除所有散落的 rgba 值：

```css
:root {
  --border-light:   rgba(125, 125, 125, 0.15);
  --border-strong:  rgba(125, 125, 125, 0.25);
  --border-faint:   rgba(125, 125, 125, 0.3);
  --scrollbar:      rgba(125, 125, 125, 0.35);
  --scrollbar-hover: rgba(125, 125, 125, 0.6);
  --code-label-bg:  rgba(125, 125, 125, 0.08);
  --inline-code-bg: rgba(125, 125, 125, 0.12);
}
```

替换范围覆盖：滚动条、代码块边框、语言标签背景、行内代码、引用块、目录侧线、移动端菜单边框，以及 JSX 中的 inline style（照片页空状态边框、文章导航分隔线）。

### Giscus 初始化修复

初始化 effect 的依赖从 `[locale, theme]` 改为 `[locale]`，并在初始化时直接读取 DOM 的实际主题状态：

```typescript
const isDark = document.documentElement.classList.contains("dark");
const giscusTheme = isDark ? "dark" : "light";
```

这样 Giscus iframe 首次加载就能拿到正确的主题，不再闪烁。

### 可访问性补齐

- 汉堡菜单按钮添加 `aria-controls="mobile-menu"`，菜单添加对应 `id`
- 标签筛选的激活项添加 `aria-current="true"`
- Lightbox 改为传递原图的 `alt` 文本，而非写死的 `"Enlarged preview"`
- 搜索框添加 `focus:ring-current`，确保焦点环可见

### 冗余代码清理

- 移除 `blog/[slug]/page.tsx` 中多余的 `dynamic = 'force-static'`
- mailto 链接移除不必要的 `target="_self"`
- RSS feed description 从混合语言改为纯中文

## 第七轮：遗漏修复与精简

第三轮深度审查，发现了几个之前遗漏的问题。

### SearchBar 遗漏的裸 rgba 值

设计令牌迁移时漏掉了 `SearchBar.tsx` 中的 `rgba(125, 125, 125, 0.05)` 背景色。新增 `--search-bg` 令牌统一管理。

### 分页链接多余的 `?`

`blog/page.tsx` 中 `baseHref` 在没有筛选参数时返回 `"?"` 而非空字符串，导致分页链接出现多余的 `?`（如 `/blog?`）。修正为空字符串，`Pagination` 组件会自动处理 `?page=2` 的拼接。

### Giscus 语言切换失效

`Comments.tsx` 用 `initialized` ref 阻止重复初始化，但这也阻止了切换语言时重新创建 Giscus iframe。移除 `initialized` 守卫，改为在 locale 变化时清空容器并重新注入 script：

```typescript
useEffect(() => {
  const container = ref.current;
  if (!container) return;
  container.innerHTML = "";
  // ... 重新创建 script
}, [locale]);
```

### CSS 变量去重

`.dark` 块中有 11 个 CSS 变量和 `:root` 完全相同（所有 rgba 系列令牌和 `--star`、`--success`）。这些值在明暗模式下不变，重复声明只是噪音。清理后 `.dark` 只保留 6 个真正需要覆盖的颜色变量。

### RSS 描述语言修正

RSS feed 的 `<description>` 从纯中文改为英文（`"Jimmy's personal blog — mobile development, tech, and more"`），因为 feed 包含中英文两种语言的文章，用英文描述更通用。

## 总结

7 轮迭代，45+ 项改进，涉及 20+ 个文件。主要收获：

- **安全性**：添加了响应头、禁用了 X-Powered-By、OG 参数校验
- **无障碍**：Lightbox 焦点陷阱、滚动锁定、aria 属性、焦点环可见性
- **性能**：MutationObserver 替代 setTimeout、Giscus postMessage、视频条件加载
- **一致性**：18 个 CSS 设计令牌、所有 UI 文案走 i18n、环境变量集中管理
- **可维护性**：数据文件抽离、类型安全、错误日志、冗余代码清理

代码审查不只是找问题，更重要的是建立规范。当项目有了统一的设计令牌、i18n 覆盖和错误处理模式，后续开发就能自然地遵循这些约定。
