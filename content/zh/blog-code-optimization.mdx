## 第一轮：高优先级修复

### 环境变量集中管理

之前 `robots.ts`、`sitemap.ts`、`feed.xml/route.ts`、`layout.tsx`、`blog/[slug]/page.tsx` 五个文件各自写了一遍 `process.env.NEXT_PUBLIC_SITE_URL || "https://example.com"`。

新建 `lib/env.ts`，抽取 `getSiteUrl()` 函数，统一处理 fallback 并在缺失时输出 `console.warn`：

```typescript
export function getSiteUrl(): string {
  const url = process.env.NEXT_PUBLIC_SITE_URL;
  if (!url) {
    console.warn("[env] NEXT_PUBLIC_SITE_URL is not set — falling back to", FALLBACK_URL);
    return FALLBACK_URL;
  }
  return url;
}
```

### Lightbox 无障碍访问

原来的 Lightbox 只是一个 `div` 模拟的弹窗，键盘用户可以 Tab 跳出去。修复后：

- 添加 `aria-modal="true"`
- 打开时自动聚焦关闭按钮
- 锁定 `body` 滚动
- 实现焦点陷阱（Tab 循环在 lightbox 内部）

### CSS 清理

- 代码块背景色 `#fafafa` / `#161b22` → `var(--code-bg)`
- 代码文字色 `#0a0a0a` / `#f0f0f0` → `var(--fg-deeper)` / `var(--fg)`
- 给所有必要的 `!important` 加上注释说明原因

### 其他修复

- OG 图片 title 截断到 100 字符
- MDX 解析错误加 `console.warn` 日志
- 硬编码的"全部"/"上一篇"/"下一篇"移入 i18n messages
- RSS `language` 从 `"zh-en"` 修正为 `"zh"`
- 布局 metadata description 改为 SEO 友好的英文
- `CodeCopyButton` 的 `setTimeout` 替换为 `MutationObserver`
- `ThemedVideo` 改为双 video 元素 + CSS visibility 切换
- SKILLS 数据从 about 页面抽离到 `data/skills.ts`
- i18n 类型断言改用 `(typeof routing.locales)[number]`

## 第六轮：深层清理与可访问性

第二轮审查后发现的问题，集中在 CSS 变量覆盖不彻底、可访问性遗漏和冗余代码三个方面。

### CSS 变量体系补全

`globals.css` 中仍有 15+ 处裸写的 `rgba(125, 125, 125, ...)` 值。新增 7 个设计令牌，彻底消除所有散落的 rgba 值：

```css
:root {
  --border-light:   rgba(125, 125, 125, 0.15);
  --border-strong:  rgba(125, 125, 125, 0.25);
  --border-faint:   rgba(125, 125, 125, 0.3);
  --scrollbar:      rgba(125, 125, 125, 0.35);
  --scrollbar-hover: rgba(125, 125, 125, 0.6);
  --code-label-bg:  rgba(125, 125, 125, 0.08);
  --inline-code-bg: rgba(125, 125, 125, 0.12);
}
```

替换范围覆盖：滚动条、代码块边框、语言标签背景、行内代码、引用块、目录侧线、移动端菜单边框，以及 JSX 中的 inline style（照片页空状态边框、文章导航分隔线）。

### Giscus 初始化修复

初始化 effect 的依赖从 `[locale, theme]` 改为 `[locale]`，并在初始化时直接读取 DOM 的实际主题状态：

```typescript
const isDark = document.documentElement.classList.contains("dark");
const giscusTheme = isDark ? "dark" : "light";
```

这样 Giscus iframe 首次加载就能拿到正确的主题，不再闪烁。

### 可访问性补齐

- 汉堡菜单按钮添加 `aria-controls="mobile-menu"`，菜单添加对应 `id`
- 标签筛选的激活项添加 `aria-current="true"`
- Lightbox 改为传递原图的 `alt` 文本，而非写死的 `"Enlarged preview"`
- 搜索框添加 `focus:ring-current`，确保焦点环可见

### 冗余代码清理

- 移除 `blog/[slug]/page.tsx` 中多余的 `dynamic = 'force-static'`
- mailto 链接移除不必要的 `target="_self"`
- RSS feed description 从混合语言改为纯中文

## 第七轮：遗漏修复与精简

第三轮深度审查，发现了几个之前遗漏的问题。

### SearchBar 遗漏的裸 rgba 值

设计令牌迁移时漏掉了 `SearchBar.tsx` 中的 `rgba(125, 125, 125, 0.05)` 背景色。新增 `--search-bg` 令牌统一管理。

### 分页链接多余的 `?`

`blog/page.tsx` 中 `baseHref` 在没有筛选参数时返回 `"?"` 而非空字符串，导致分页链接出现多余的 `?`（如 `/blog?`）。修正为空字符串，`Pagination` 组件会自动处理 `?page=2` 的拼接。

### Giscus 语言切换失效

`Comments.tsx` 用 `initialized` ref 阻止重复初始化，但这也阻止了切换语言时重新创建 Giscus iframe。移除 `initialized` 守卫，改为在 locale 变化时清空容器并重新注入 script：

```typescript
useEffect(() => {
  const container = ref.current;
  if (!container) return;
  container.innerHTML = "";
  // ... 重新创建 script
}, [locale]);
```

### CSS 变量去重

`.dark` 块中有 11 个 CSS 变量和 `:root` 完全相同（所有 rgba 系列令牌和 `--star`、`--success`）。这些值在明暗模式下不变，重复声明只是噪音。清理后 `.dark` 只保留 6 个真正需要覆盖的颜色变量。

### RSS 描述语言修正

RSS feed 的 `<description>` 从纯中文改为英文（`"Jimmy's personal blog — mobile development, tech, and more"`），因为 feed 包含中英文两种语言的文章，用英文描述更通用。

## 第八轮：架构优化与 i18n 补全

这一轮聚焦三个方面：代码复用、国际化覆盖、SEO。

### 提取 `useDarkMode` Hook

`ThemedVideo.tsx` 和 `Comments.tsx` 各自创建了一个 MutationObserver 监听 `document.documentElement` 的 class 变化来追踪暗色模式状态，逻辑完全相同。提取为 `hooks/useDarkMode.ts`：

```typescript
export function useDarkMode(): boolean {
  const [dark, setDark] = useState(false);
  useEffect(() => {
    const sync = () =>
      setDark(document.documentElement.classList.contains("dark"));
    sync();
    const observer = new MutationObserver(sync);
    observer.observe(document.documentElement, {
      attributes: true,
      attributeFilter: ["class"],
    });
    return () => observer.disconnect();
  }, []);
  return dark;
}
```

两个组件各减少约 15 行样板代码。`ThemeToggle` 保留自己的逻辑，因为它还需要写入 localStorage 和应用 class。

### UI 组件 i18n 补全

四个客户端组件的交互文案之前是硬编码英文：

- `ThemeToggle` — "Switch to light/dark mode"
- `BackToTop` — "Back to top"
- `CodeCopyButton` — "Copy" / "Copied!" / "Failed"
- `LanguageSwitcher` — "Toggle language"

新增 `a11y` 命名空间，统一管理这些无障碍和交互文案：

```json
{
  "a11y": {
    "lightMode": "切换到浅色模式",
    "darkMode": "切换到深色模式",
    "backToTop": "回到顶部",
    "copyCode": "复制代码",
    "copied": "已复制！",
    "copyFailed": "复制失败",
    "toggleLang": "切换语言"
  }
}
```

### SEO：canonical 和 alternate URL

之前没有设置 `alternates` 元数据，搜索引擎可能把 `/zh/blog/xxx` 和 `/en/blog/xxx` 当作重复内容。在根布局和博客文章页面都添加了 canonical 和 alternate 链接：

```typescript
alternates: {
  canonical: `/${locale}/blog/${slug}`,
  languages: { zh: `/zh/blog/${slug}`, en: `/en/blog/${slug}` },
},
```

### TableOfContents 优化

MutationObserver 之前在每次 DOM 变化时都会重新初始化 IntersectionObserver（包括 CodeCopyButton 插入按钮时）。添加了 heading ID 缓存，只在标题实际变化时才重建：

```typescript
const key = items.map((h) => h.id).join(",");
if (key === prevKeyRef.current) return;
prevKeyRef.current = key;
```

## 第九轮：i18n 死角清理

这一轮聚焦的是之前遗漏的 i18n 死角——那些藏在工具函数和数据文件里的硬编码文案。

### 阅读时间 i18n

`lib/reading-time.ts` 直接返回格式化后的字符串 `"X 分钟阅读"` / `"X min read"`，绕过了翻译系统。重构为只返回分钟数，由调用方通过 `t("readingTime", { minutes })` 格式化：

```typescript
// Before: return locale === "zh" ? `${rounded} 分钟阅读` : `${rounded} min read`;
// After:  return Math.max(1, Math.ceil(minutes));
```

新增 i18n key：`blog.readingTime` → `"{minutes} 分钟阅读"` / `"{minutes} min read"`。

### 书单笔记双语化

`data/books.ts` 的书评笔记只有中文，英文用户看到的是中文笔记。将 `note` 字段从 `string` 改为 `Record<"zh" | "en", string>`，和 `projects.ts` 的 `description` 保持一致。

### Stars 组件 aria-label

`books/page.tsx` 的星级评分组件 `aria-label` 写死了英文 `"X stars"`。新增 `books.stars` i18n key，改为 `t("stars", { count })`。

### 照片 alt 文本边界修复

`photos/page.tsx` 的 `altFromPath()` 用 `??` 做 fallback，但 `??` 只捕获 `null`/`undefined`，不捕获空字符串。极端情况下（如文件名只有扩展名）会返回空 alt。改为 `||` 确保 fallback 生效。

## 第十轮：无障碍与分页优化

第十轮审查聚焦在无障碍遗漏、分页组件的可扩展性，以及散落的硬编码文案。

### LanguageSwitcher aria-label 修复

`LanguageSwitcher.tsx` 的 `aria-label` 写死了 `"Switch to English"` / `"切换到中文"`，但组件已经通过 `useTranslations("a11y")` 拿到了 `t("toggleLang")`。直接复用即可，消除硬编码。

### 汉堡菜单 aria-label i18n

`Navigation.tsx` 的汉堡按钮 `aria-label` 写死了 `"Close menu"` / `"Open menu"`。新增 `a11y.openMenu` 和 `a11y.closeMenu` 两个 i18n key，改为 `ta("closeMenu")` / `ta("openMenu")`。

### 分页组件省略号

原来的 `Pagination.tsx` 渲染所有页码，文章多了会产生大量 DOM 节点。重构为省略号模式：页数超过 7 时，只显示当前页附近的页码，其余用 `…` 代替。同时给每个页码链接添加 `aria-label="Page N"`。

```typescript
function getPageRange(current: number, total: number): (number | "...")[] {
  if (total <= 7) return Array.from({ length: total }, (_, i) => i + 1);
  const pages: (number | "...")[] = [1];
  if (current > 3) pages.push("...");
  const start = Math.max(2, current - 1);
  const end = Math.min(total - 1, current + 1);
  for (let i = start; i <= end; i++) pages.push(i);
  if (current < total - 2) pages.push("...");
  pages.push(total);
  return pages;
}
```

### TOC aria-current

`TableOfContents.tsx` 的激活链接只用 CSS class 标记，屏幕阅读器无法感知。添加 `aria-current="location"` 属性。

### Lightbox 动态 aria-label

Lightbox 的 `aria-label` 写死了 `"Image preview"`，改为使用实际图片的 `alt` 文本，让屏幕阅读器能准确描述弹窗内容。

### RSS lastBuildDate

RSS feed 缺少 `lastBuildDate` 元素，补充后符合 RSS 2.0 规范。

### books.ts 乱码修复

`data/books.ts` 中 "虽然有些过时" 的 "些" 字因编码问题变成了乱码（Unicode 替换字符），修复为正确的中文。

## 第十一轮：测试修复、导航无障碍与类型安全

这一轮发现了两个关键 bug：测试套件完全失效，以及多处无障碍和类型安全遗漏。

### 阅读时间测试修复

`lib/__tests__/reading-time.test.ts` 的所有断言都期望 `getReadingTime()` 返回格式化字符串（如 `"1 分钟阅读"`），但第九轮重构后函数已改为返回数字。9 个测试全部失效。修正所有断言为数字比较。

### 文章测试修复

`lib/__tests__/posts.test.ts` 引用了一个不存在的 `hello-world` 测试文章，导致 3 个测试失败。改为使用实际存在的 `building-my-blog` 文章，并更新对应的 frontmatter 断言。

### 导航链接 aria-current

`Navigation.tsx` 的桌面端和移动端导航链接只用 CSS class `active` 标记当前页面，屏幕阅读器无法感知。添加 `aria-current="page"` 属性。

### CodeCopyButton aria-label 同步

复制按钮在创建时设置了 `aria-label`，但状态变化（"已复制！"/"复制失败"）时没有同步更新 `aria-label`。屏幕阅读器用户无法听到状态反馈。修复为每次状态变化时同步更新。

### ThemedVideo aria-hidden

首页背景视频是纯装饰性的，但缺少 `aria-hidden="true"`，屏幕阅读器可能会不必要地播报。两个 video 元素都添加了该属性。

### "全部"标签链接多余的 ?

博客页面的"全部"标签在没有搜索参数时链接到 `?`，产生多余的 `?` 后缀。修正为空字符串。

### Locale 类型安全

`projects/page.tsx` 和 `books/page.tsx` 使用 `locale as "zh" | "en"` 硬编码类型断言。新增 `Locale` 类型导出到 `i18n/routing.ts`，改为 `locale as Locale`，后续新增语言时只需改一处。

## 总结

11 轮迭代，70+ 项改进，涉及 20+ 个文件。主要收获：

- **安全性**：添加了响应头、禁用了 X-Powered-By、OG 参数校验
- **无障碍**：Lightbox 焦点陷阱、滚动锁定、aria 属性、焦点环可见性、UI 文案 i18n
- **性能**：MutationObserver 替代 setTimeout、Giscus postMessage、视频条件加载、TOC 缓存优化
- **一致性**：18 个 CSS 设计令牌、所有 UI 文案走 i18n、环境变量集中管理、数据文件双语化
- **可维护性**：useDarkMode Hook、数据文件抽离、类型安全、错误日志、冗余代码清理
- **SEO**：canonical/alternate URL、结构化数据、RSS 规范化

代码审查不只是找问题，更重要的是建立规范。当项目有了统一的设计令牌、i18n 覆盖和错误处理模式，后续开发就能自然地遵循这些约定。
