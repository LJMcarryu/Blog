## 前言

这个系列前面七篇主要面向开发者——[写 PRD](/zh/blog/ai-prd)、[做设计](/zh/blog/ai-ui-design)、[写代码](/zh/blog/ai-product-dev-pipeline)、[选工具](/zh/blog/ai-tool-selection)。但 AI 的价值远不止写代码。

如果你是产品经理、设计师、运营或市场人员，AI 同样能大幅提升你的工作效率。你不需要懂编程，不需要了解技术细节，只需要学会"怎么跟 AI 说话"。

这篇文章覆盖非技术岗位最常见的工作场景，每个场景给出：能用什么工具、怎么写 Prompt、输出是什么样、有什么局限。读完就能用。

如果你对 Prompt 技巧感兴趣，推荐搭配 [Prompt 工程实战手册](/zh/blog/ai-prompt-engineering) 一起读——那篇讲的技巧（角色设定、模板约束、多轮迭代）在这里全部适用。

## 一、工具选择：非技术人员用什么

不需要复杂的开发工具。以下三个就够了：

| 工具 | 价格 | 适合场景 | 推荐理由 |
|------|------|---------|---------|
| Claude | 免费 / $20月 | 长文档、结构化输出、分析 | 格式精确，中文好，上下文长 |
| ChatGPT | 免费 / $20月 | 日常问答、头脑风暴、创意 | 生态丰富，上手简单 |
| Gemini | 免费 / $20月 | 需要搜索真实数据的任务 | 搜索集成好，能引用来源 |

**建议**：先用免费版试一周，确认哪个顺手再考虑付费。大部分场景免费版就够用。

详细的工具对比参考 [AI 工具选型指南](/zh/blog/ai-tool-selection)。

## 二、竞品调研

### 场景

老板说"下周一之前给我一份竞品分析报告"。传统做法：花两天搜集资料、整理表格、写分析。AI 辅助：半天搞定初稿。

### 方法

**第一步：让 AI 生成调研框架**

```
我是一名产品经理，需要做工单系统的竞品分析。
请帮我生成一份竞品调研框架，包括：
1. 需要调研的维度（功能、定价、目标用户、优劣势）
2. 主流竞品列表（至少 5 个）
3. 对比表格的模板

注意：如果你不确定某个信息，请标注 [需验证]，不要编造数据。
```

**第二步：逐个竞品填充信息**

用 Gemini（联网搜索）逐个调研：

```
请搜索 Zendesk 的最新信息，填充以下维度：
- 核心功能
- 定价方案（列出具体价格）
- 目标用户群体
- 最近一年的重大更新
- 用户评价中的常见优点和缺点

请引用信息来源。
```

**第三步：生成分析结论**

```
基于以上 5 个竞品的信息，请分析：
1. 市场空白点（哪些需求没有被很好满足）
2. 定价策略建议（我们应该定在什么价位）
3. 差异化方向（我们应该在哪些方面做出差异）

请用表格对比，最后给出 3 条可执行的建议。
```

### 局限性

- **数据准确性**：AI 生成的具体数字（市场份额、用户量、营收）必须人工验证
- **时效性**：AI 的知识有截止日期，最新的产品更新可能没有覆盖
- **深度不足**：AI 能给你"面"上的信息，但"点"上的深度洞察（用户访谈、实际体验）需要你自己做

## 三、写周报和工作总结

### 场景

每周五下午最痛苦的事：写周报。你知道这周做了什么，但组织成文字就是费��。

### 方法

**把零散信息丢给 AI，让它帮你组织**：

```
请帮我整理本周工作周报。以下是我这周做的事情（零散记录）：

- 周一：和开发对了工单系统的需求，改了 3 个字段
- 周一下午：写了通知模块的 PRD 初稿
- 周二：竞品调研，看了 Zendesk 和 Freshdesk
- 周三：PRD 评审会，收到 5 条反馈，改了优先级排序逻辑
- 周四：和设计师对了 UI，确认了列表页和详情页
- 周五上午：写了下周的迭代计划

请按以下格式输出：
## 本周完成
[按项目分组，每项一句话总结]

## 关键进展
[最重要的 2-3 个进展]

## 下周计划
[基于本周工作推导]

## 需要协助
[如果有的话]

风格要求：简洁专业，每条不超过两句话。
```

### 进阶：用 AI 写月度/季度总结

月度总结需要更高的抽象层次。把四周的周报丢给 AI：

```
以下是我过去四周的周报。请帮我写一份月度工作总结：

1. 提炼本月的核心成果（3-5 条）
2. 量化成果（如果有数据的话）
3. 遇到的挑战和解决方案
4. 下月重点方向

风格：面向直属领导，突出价值和影响力。

[粘贴四周周报]
```

### 局限性

- AI 不知道你工作的上下文（团队目标、OKR、领导关注点），需要你补充
- 不要让 AI 编造你没做的事情——它会"合理推测"你可能做了什么

## 四、整理会议纪要

### 场景

开了一小时的会，讨论了十几个话题，你需要整理出一份结构化的会议纪要。

### 方法

**如果有录音/转录文本**：

```
以下是今天产品评审会的转录文本。请整理成会议纪要：

格式要求：
## 会议信息
- 日期：[填写]
- 参会人：[从转录中提取]
- 主题：[总结]

## 讨论要点
[按话题分组，每个话题列出：讨论内容摘要、不同意见、最终结论]

## 决策事项
[明确的决策，标注决策人]

## 待办事项
[每条标注：负责人、截止日期]

## 遗留问题
[未达成共识的问题]

注意：
- 如果转录中某段话不清楚，标注 [不确定]
- 保留关键的原话引用
- 待办事项必须有明确的负责人

[粘贴转录文本]
```

**如果只有你的笔记**：

```
以下是我在会议中的速记。请帮我整理成正式的会议纪要：

[粘贴速记]

要求：
- 补充逻辑连接，让速记变成完整的句子
- 按话题分组
- 提取所有 action items
- 不要添加我没有记录的内容
```

### 局限性

- AI 可能误解速记中的缩写和上下文
- 转录文本中的同音词可能被错误识别
- 始终让参会人确认纪要的准确性

## 五、分析用户反馈

### 场景

你有 200 条用户反馈（来自客服工单、应用商店评论、用户访谈），需要找出共性问题和改进方向。

### 方法

**第一步：分类和标签**

```
以下是 50 条用户反馈。请对每条反馈进行分类：

分类维度：
1. 反馈类型：功能请求 / Bug 报告 / 体验问题 / 正面评价 / 其他
2. 涉及模块：工单创建 / 工单列表 / 通知 / 搜索 / 移动端 / 其他
3. 情感倾向：正面 / 中性 / 负面
4. 紧急程度：高（影响核心流程）/ 中 / 低

请用表格输出，每行一条反馈。

[粘贴反馈]
```

**第二步：统计和洞察**

```
基于上面的分类结果，请分析：

1. 各类型反馈的数量占比（饼图描述）
2. 被提及最多的 Top 5 问题
3. 负面反馈中的共性模式
4. 用户最期待的 Top 3 功能
5. 需要紧急处理的问题列表

请用数据支撑每个结论。
```

**第三步：生成行动建议**

```
基于以上分析，请给出产品改进建议：

1. 短期（本月可做）：修复哪些问题
2. 中期（本季度）：优化哪些体验
3. 长期（下季度规划）：开发哪些新功能

每条建议标注：预期影响（高/中/低）、实施难度（高/中/低）。
按"高影响+低难度"优先排序。
```

### 局限性

- AI 的分类可能不准确，特别是模糊的反馈（"不好用"到底是哪里不好用）
- 定量分析（占比、排名）基于 AI 的分类结果，如果分类有误，数据也会有偏差
- 深层用户需求需要结合用户访谈，不能只靠文本分析

## 六、生成数据报告

### 场景

你有一份 Excel 数据（用户增长、功能使用率、转化漏斗），需要写一份给领导看的数据分析报告。

### 方法

**第一步：描述数据**

```
以下是我们工单系统上线一个月的数据：

- 总工单数：1,247
- 日均工单：42
- 平均处理时间：4.2 小时
- 首次响应时间：23 分钟
- 用户满意度：4.1/5
- 按状态分布：已完成 68%，处理中 22%，待分配 10%
- 按优先级：低 45%，中 35%，高 15%，紧急 5%
- 按部门：IT 40%，行政 25%，财务 20%，其他 15%

请生成一份数据分析报告，面向管理层。
```

**第二步：指定报告结构**

```
报告结构：

## 摘要
[3 句话总结核心发现]

## 关键指标
[用表格展示，标注同比/环比变化趋势，用 ↑↓→ 表示]

## 亮点
[做得好的 2-3 个方面，用数据支撑]

## 需要关注
[需要改进的 2-3 个方面，用数据支撑]

## 建议
[基于数据的 3 条可执行建议]

风格：简洁、数据驱动、面向非技术管理层。避免技术术语。
```

### 局限性

- AI 不能直接读取 Excel 文件（除非你粘贴数据或用支持文件上传的版本）
- AI 可能过度解读数据——相关性不等于因果性
- 关键的业务判断（这个数据好不好、目标是什么）需要你自己给出

## 七、写营销文案和公众号文章

### 场景

需要写一篇产品发布的公众号文章，或者一系列社交媒体推广文案。

### 方法

**产品发布文章**：

```
请帮我写一篇工单系统上线的公众号文章。

产品信息：
- 产品名：SmartTicket 智能工单系统
- 核心卖点：AI 自动分类、智能分配、实时通知
- 目标读者：企业 IT 管理者
- 上线日期：2026 年 4 月 1 日

文章要求：
- 标题：吸引点击但不标题党，给出 3 个备选
- 开头：用一个 IT 管理者的痛点场景引入
- 中间：介绍 3 个核心功能，每个配一个使用场景
- 结尾：引导试用，附注册链接占位符
- 字数：800-1000 字
- 风格：专业但不枯燥，适度使用比喻

不要使用以下词汇：颠覆、革命性、划时代、赋能
```

**社交媒体文案**：

```
基于上面的产品信息，请生成以下平台的推广文案：

1. 微信朋友圈（150 字以内，配图文案）
2. 小红书（300 字，带话题标签，口语化）
3. LinkedIn（200 字，专业风格，英文）
4. Twitter/X（280 字符以内，英文）

每个平台生成 2 个版本供选择。
```

### 局限性

- AI 生成的文案往往"正确但平庸"——缺少品牌个性和情感共鸣
- 需要你注入品牌调性和独特视角
- 数据和案例需要用真实的，不要用 AI 编造的

## 八、日常提效小技巧

### 邮件润色

```
请帮我润色这封邮件，要求：
- 语气：专业友好
- 保持原意，优化表达
- 如果有语法错误请修正
- 控制在 200 字以内

[粘贴邮件草稿]
```

### 翻译 + 本地化

```
请将以下中文翻译成英文，注意：
- 不是逐字翻译，而是本地化表达
- 保持专业但不生硬
- 产品名称保留原文

[粘贴中文]
```

### 头脑风暴

```
我在策划工单系统的用户增长方案。请帮我头脑风暴：

1. 先给出 10 个增长策略的方向
2. 每个方向用一句话描述
3. 标注可行性（高/中/低）和预期效果（高/中/低）
4. 最后推荐 Top 3 并说明理由

约束：我们是 B2B 产品，预算有限，团队 5 人。
```

### 制作 PPT 大纲

```
我需要做一个 15 分钟的产品汇报 PPT，面向公司高管。

内容要点：
- 工单系统上线一个月的成果
- 关键数据指标
- 用户反馈摘要
- 下阶段计划

请生成 PPT 大纲：
- 每页的标题和要点（3-4 个要点/页）
- 建议的页数：8-10 页
- 标注哪些页需要图表、哪些页需要截图
- 开头和结尾的设计建议
```

## 九、避坑提醒

非技术人员使用 AI 时，最常踩的坑：

1. **把 AI 输出当事实**：AI 生成的数据、引用、案例可能是编造的。所有关键信息必须验证。详见 [踩坑指南](/zh/blog/ai-dev-pitfalls)

2. **一次性要求太多**：不要在一个 Prompt 里塞十个要求。拆成多轮，每轮聚焦一个方面，效果好得多

3. **不给上下文**：AI 不知道你的公司、团队、目标。给的背景信息越多，输出越有针对性

4. **直接用不改**：AI 的输出是初稿，不是终稿。你的专业判断、行业经验、对用户的理解——这些是 AI 没有的

5. **忽略隐私**：不要把公司机密数据、客户个人信息直接粘贴给 AI。使用脱敏数据，或确认你使用的工具不会用你的数据训练模型

## 十、总结

AI 对非技术人员的价值可以用一句话概括：**消除重复性工作，让你把时间花在判断和决策上**。

三个核心建议：

1. **从最痛的场景开始**。如果你每周花 2 小时写周报，先用 AI 优化这个环节。立刻见效，建立信心
2. **学会写好 Prompt**。非技术人员最大的杠杆点不是选工具，而是学会"怎么跟 AI 说话"。读一遍 [Prompt 手册](/zh/blog/ai-prompt-engineering)，投入一小时，回报持续
3. **AI 是助手，你是专家**。AI 帮你更快地完成执行层面的工作，但业务判断、用户洞察、战略思考——这些是你的不可替代价值
