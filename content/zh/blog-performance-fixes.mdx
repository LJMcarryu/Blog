## 背景

在完成博客的功能开发后，我在生产构建时发现了一个警告：博客文章页面被标记为"动态渲染"。这意味着每次用户访问都需要服务器实时渲染，而不是使用预先生成的静态 HTML。

对于一个内容型博客来说，这是性能和成本的双重浪费。这篇文章记录了我如何解决这个问题，以及顺带修复的其他几个关键 bug。

## 问题 1：博客文章被动态渲染

### 现象

运行 `npm run build` 时看到警告：

```
The path /en/blog/blog-iteration-journey is marked as "dynamic"
since it will be rendered for each user at request time.
```

### 原因分析

虽然我已经实现了 `generateStaticParams()`，但 Next.js 仍然选择动态渲染。常见原因包括：

1. 使用了动态 API（`cookies()`, `headers()`, `searchParams`）
2. fetch 使用了 `cache: 'no-store'`
3. 没有明确告诉 Next.js 强制静态生成

### 解决方案

在 `app/[locale]/blog/[slug]/page.tsx` 中添加两个配置：

```typescript
export async function generateStaticParams() {
  const locales = ["zh", "en"];
  const params: { locale: string; slug: string }[] = [];

  for (const locale of locales) {
    const posts = getPostsByLocale(locale);
    posts.forEach((post) => {
      params.push({ locale, slug: post.slug });
    });
  }

  return params;
}

// ✅ 新增：强制静态生成
export const dynamicParams = false; // 404 for unknown slugs
export const dynamic = 'force-static'; // Force static generation
```

**配置说明：**

- `dynamicParams = false`：对于不在 `generateStaticParams()` 列表中的 slug，直接返回 404，而不是尝试动态渲染
- `dynamic = 'force-static'`：强制静态生成，如果页面使用了动态 API 会在构建时报错（帮助发现问题）

### 效果

构建输出从：

```
ƒ /[locale]/blog/[slug]  (Dynamic)
```

变为：

```
● /[locale]/blog/[slug]  (SSG)
  ├ /zh/blog/blog-iteration-journey
  ├ /zh/blog/building-my-blog
  └ [+10 more paths]
```

所有博客文章在构建时预渲染为静态 HTML，带来：

- **更快的页面加载**：无需服务器渲染
- **更好的 SEO**：爬虫直接获取完整 HTML
- **更低的成本**：减少服务器计算资源

## 问题 2：TableOfContents 内存泄漏

### 现象

浏览器控制台报错：

```
TypeError: Cannot create property 'observer' on number '136'
```

### 原因分析

查看 `components/TableOfContents.tsx` 的代码：

```typescript
useEffect(() => {
  const timer = setTimeout(() => {
    // ... 创建 observer
    const observer = new IntersectionObserver(/* ... */);

    // ❌ 错误：试图在数字上创建属性
    (timer as unknown as { observer: IntersectionObserver }).observer = observer;
  }, 150);

  return () => {
    clearTimeout(timer);
    // ❌ 没有清理 observer
  };
}, []);
```

问题在于：

1. `setTimeout` 返回的 `timer` 是一个数字（timer ID）
2. 试图在数字上创建 `observer` 属性会报错
3. cleanup 函数没有断开 IntersectionObserver，导致内存泄漏

### 解决方案

将 `observer` 提升到 `useEffect` 作用域：

```typescript
useEffect(() => {
  let observer: IntersectionObserver | null = null;

  const timer = setTimeout(() => {
    const items = getHeadings();
    if (items.length < 2) return;

    setHeadings(items);

    // ✅ 直接赋值给作用域变量
    observer = new IntersectionObserver(
      (entries) => {
        for (const entry of entries) {
          if (entry.isIntersecting) {
            setActiveId(entry.target.id);
          }
        }
      },
      { rootMargin: "0px 0px -70% 0px", threshold: 0 }
    );

    const article = document.querySelector("article");
    article?.querySelectorAll("h2, h3").forEach((node) => observer?.observe(node));
  }, 150);

  return () => {
    clearTimeout(timer);
    // ✅ 正确清理 observer
    observer?.disconnect();
  };
}, []);
```

**改进点：**

1. `observer` 作为 `useEffect` 作用域变量，可以在 cleanup 中访问
2. cleanup 函数正确调用 `disconnect()` 断开所有观察
3. 避免了在原始类型上创建属性的错误

## 问题 3：样式和主题优化

### 颜色对比度改进

原来的颜色对比度不够，影响可读性。调整了设计 token：

```css
/* 之前 */
:root {
  --fg: #555555;        /* 浅灰 */
  --fg-deep: #222222;
  --fg-light: #888888;
}

.dark {
  --c-bg: #050505;      /* 接近黑 */
  --fg: #bbbbbb;        /* 浅灰 */
}

/* 之后 */
:root {
  --fg: #333333;        /* 更深，对比度更高 */
  --fg-deep: #111111;
  --fg-light: #666666;
}

.dark {
  --c-bg: #000000;      /* 纯黑背景 */
  --fg: #e0e0e0;        /* 更亮，对比度更高 */
  --fg-deep: #eeeeee;
  --fg-light: #a8a8a8;
}
```

### 代码块语言标签

为代码块添加语言标签，方便读者识别：

```css
.prose pre > code[data-language]::before {
  content: attr(data-language);
  position: absolute;
  top: 0;
  left: 0;
  padding: 0.15rem 0.6rem;
  font-size: 0.65rem;
  text-transform: uppercase;
  letter-spacing: 0.05em;
  color: var(--fg-light);
  opacity: 0.6;
  background: rgba(125, 125, 125, 0.08);
  border-bottom-right-radius: 0.375rem;
}
```

效果：代码块左上角显示 `TYPESCRIPT`、`BASH` 等标签。

### Giscus 评论主题同步

之前评论系统使用 `preferred_color_scheme`，但切换主题时不会实时更新。改为监听主题变化：

```typescript
export default function Comments() {
  const [theme, setTheme] = useState<"light" | "dark">("light");

  // 监听主题变化
  useEffect(() => {
    const updateTheme = () => {
      const isDark = document.documentElement.classList.contains("dark");
      setTheme(isDark ? "dark" : "light");
    };

    updateTheme();

    const observer = new MutationObserver(updateTheme);
    observer.observe(document.documentElement, {
      attributes: true,
      attributeFilter: ["class"],
    });

    return () => observer.disconnect();
  }, []);

  useEffect(() => {
    // ... 重新加载 Giscus 脚本
    script.setAttribute("data-theme", theme);
    // ...
  }, [locale, theme]); // ✅ 依赖 theme

  return <div ref={ref} />;
}
```

现在切换主题时，评论区会立即重新加载并应用新主题。

## 总结

这次优化解决了三个关键问题：

1. **性能优化**：强制静态生成，所有博客文章预渲染为 HTML
2. **内存泄漏修复**：正确清理 IntersectionObserver
3. **用户体验改进**：更好的颜色对比度、代码块标签、主题同步

这些看似小的改进，累积起来对用户体验有显著提升。性能优化不仅仅是追求数字，更重要的是让用户感受到"快"和"流畅"。
