## Round 1: High-Priority Fixes

### Centralized Environment Variables

Five files each had their own copy of `process.env.NEXT_PUBLIC_SITE_URL || "https://example.com"` — `robots.ts`, `sitemap.ts`, `feed.xml/route.ts`, `layout.tsx`, and `blog/[slug]/page.tsx`.

Created `lib/env.ts` with a `getSiteUrl()` function that handles the fallback and warns when the env var is missing:

```typescript
export function getSiteUrl(): string {
  const url = process.env.NEXT_PUBLIC_SITE_URL;
  if (!url) {
    console.warn("[env] NEXT_PUBLIC_SITE_URL is not set — falling back to", FALLBACK_URL);
    return FALLBACK_URL;
  }
  return url;
}
```

## Round 8: Architecture and i18n Completion

This round focused on three areas: code reuse, i18n coverage, and SEO.

### Extract `useDarkMode` Hook

`ThemedVideo.tsx` and `Comments.tsx` each created their own MutationObserver to track dark mode state — identical logic. Extracted to `hooks/useDarkMode.ts`:

```typescript
export function useDarkMode(): boolean {
  const [dark, setDark] = useState(false);
  useEffect(() => {
    const sync = () =>
      setDark(document.documentElement.classList.contains("dark"));
    sync();
    const observer = new MutationObserver(sync);
    observer.observe(document.documentElement, {
      attributes: true,
      attributeFilter: ["class"],
    });
    return () => observer.disconnect();
  }, []);
  return dark;
}
```

Each component shed ~15 lines of boilerplate. `ThemeToggle` keeps its own logic since it also writes to localStorage and applies the class.

### UI Component i18n Completion

Four client components had hardcoded English interaction labels:

- `ThemeToggle` — "Switch to light/dark mode"
- `BackToTop` — "Back to top"
- `CodeCopyButton` — "Copy" / "Copied!" / "Failed"
- `LanguageSwitcher` — "Toggle language"

Added an `a11y` namespace to manage all accessibility and interaction labels:

```json
{

    "backToTop": "Back to top",
```

### SEO: Canonical and Alternate URLs

No `alternates` metadata was set, so search engines could treat `/zh/blog/xxx` and `/en/blog/xxx` as duplicate content. Added canonical and alternate links to both the root layout and blog post pages:

```typescript

if (key === prevKeyRef.current) return;
prevKeyRef.current = key;
```

## Round 9: i18n Blind Spot Cleanup

This round focused on i18n blind spots — hardcoded strings hiding in utility functions and data files.

### Reading Time i18n

`lib/reading-time.ts` returned pre-formatted strings like `"X 分钟阅读"` / `"X min read"`, bypassing the translation system. Refactored to return just the minute count, with the caller formatting via `t("readingTime", { minutes })`:

```typescript
// Before: return locale === "zh" ? `${rounded} 分钟阅读` : `${rounded} min read`;
// After:  return Math.max(1, Math.ceil(minutes));
```

Added i18n key: `blog.readingTime` → `"{minutes} min read"` / `"{minutes} 分钟阅读"`.

### Bilingual Book Notes

`data/books.ts` had Chinese-only book review notes — English users saw Chinese text. Changed `note` from `string` to `Record<"zh" | "en", string>`, matching the pattern used in `projects.ts`.

### Stars Component aria-label

The star rating component in `books/page.tsx` had a hardcoded English `aria-label` of `"X stars"`. Added `books.stars` i18n key and changed to `t("stars", { count })`.

### Photo Alt Text Edge Case

`photos/page.tsx`'s `altFromPath()` used `??` for the fallback, but `??` only catches `null`/`undefined`, not empty strings. In edge cases (e.g., a filename that's just an extension), it would return empty alt text. Changed to `||` to ensure the fallback triggers.

## Round 10: Accessibility and Pagination

This round focused on accessibility gaps, pagination scalability, and remaining hardcoded strings.

### LanguageSwitcher aria-label Fix

`LanguageSwitcher.tsx` had a hardcoded `aria-label` of `"Switch to English"` / `"切换到中文"`, despite already having `t("toggleLang")` from `useTranslations("a11y")`. Replaced with the i18n value directly.

### Hamburger Menu aria-label i18n

`Navigation.tsx`'s hamburger button had hardcoded `"Close menu"` / `"Open menu"` aria-labels. Added `a11y.openMenu` and `a11y.closeMenu` i18n keys and switched to `ta("closeMenu")` / `ta("openMenu")`.

### Pagination Ellipsis

The original `Pagination.tsx` rendered every page number — fine for 5 pages, but creates excessive DOM nodes at scale. Refactored to show an ellipsis pattern when total pages exceed 7, displaying only the current page's neighbors:

```typescript
function getPageRange(current: number, total: number): (number | "...")[] {
  if (total <= 7) return Array.from({ length: total }, (_, i) => i + 1);
  const pages: (number | "...")[] = [1];
  if (current > 3) pages.push("...");
  const start = Math.max(2, current - 1);
  const end = Math.min(total - 1, current + 1);
  for (let i = start; i <= end; i++) pages.push(i);
  if (current < total - 2) pages.push("...");
  pages.push(total);
  return pages;
}
```

Also added `aria-label="Page N"` to each page link for screen reader clarity.

### TOC aria-current

The active TOC link used a CSS class but had no `aria-current="location"` attribute. Screen readers now correctly identify the current section.

### Lightbox Dynamic aria-label

The Lightbox dialog had a static `aria-label="Image preview"`. Changed to use the actual image's `alt` text, so screen readers describe the lightbox content accurately.

### RSS lastBuildDate

The RSS feed was missing the `lastBuildDate` element. Added it for RSS 2.0 compliance.

### books.ts Encoding Fix

## Round 12: Deep Review and Type Safety Hardening

Round 12 was a comprehensive deep review covering performance, SEO, type safety, and accessibility.

### BackToTop Scroll Throttling

`BackToTop.tsx`'s scroll listener called `setVisible()` on every scroll event — potentially 60+ state updates per second. Added `requestAnimationFrame` throttling to ensure at most one update per frame:

```typescript
let ticking = false;
function onScroll() {
  if (!ticking) {
    ticking = true;
    requestAnimationFrame(() => {
      setVisible(window.scrollY > 400);
      ticking = false;
    });
  }
}
```

### CodeCopyButton type Attribute

The dynamically created copy button was missing `type="button"`, which could cause it to be treated as `type="submit"` in certain contexts. Added `btn.type = "button"`.

### RSS Feed Fixes

The RSS feed had several issues:
- `<language>` was hardcoded as `zh`, changed to `zh-CN`
- Missing `<author>` tags on items
- CDATA content didn't escape `]]>`, which could break XML parsing
- `lastBuildDate` used the current time on every request, changed to use the most recent post's publish date

### SEO Fixes

- Blog post canonical URLs were relative (`/zh/blog/slug`), which search engines may not resolve correctly. Changed to absolute URLs (`https://example.com/zh/blog/slug`)
- JSON-LD structured data was missing `dateModified`. Added it to match `datePublished`

### Global Locale Type Safety

The `locale` parameter in `lib/posts.ts`, `lib/reading-time.ts`, `data/links.ts`, `data/now.ts`, and `data/uses.ts` was typed as `string`, providing no compile-time validation. Changed all to use the `Locale` type from `i18n/routing.ts`. Also updated `data/books.ts` and `data/projects.ts` from `Record<"zh" | "en", string>` to `Record<Locale, string>`.

### CSS Cleanup

Removed unused `.slide-enter-6` and `.slide-enter-10` CSS classes and their corresponding animation selectors.

### Lightbox Focus Trap Improvement

The focus trap selector only included `button`, `[href]`, and `[tabindex]`, missing `input`, `select`, and `textarea`. While the current Lightbox doesn't contain form elements, a robust focus trap should cover all interactive elements.

## Round 13: Accessibility Hardening, Middleware Fix, and SEO Completion

Another comprehensive deep review, covering accessibility, configuration, SEO, and code robustness.

### Middleware Export Fix (Critical Bug)

`proxy.ts` exported a function named `proxy`, but Next.js requires the middleware file to be named `middleware.ts` and export `default` or `middleware`. Renamed the file and fixed the export:

```typescript
// Before: export const proxy = createMiddleware(routing);
// After:  export default createMiddleware(routing);
```

### RSS Feed Multilingual Support

The RSS feed's `<language>` tag was hardcoded to `zh-CN`, but the feed contains posts in both languages. Added per-item `<language>` tags:

```xml
<language>zh-CN</language>  <!-- Chinese posts -->
<language>en-US</language>  <!-- English posts -->
```

### CSS Accessibility Trifecta

1. **`prefers-reduced-motion`**: Added a global media query that disables all animations and transitions for users who prefer reduced motion
2. **`focus-visible` rings**: Added keyboard focus styles to 6 interactive elements — `.nav-link`, `.tag-pill`, `.pagination-link`, `.copy-btn`, `.back-to-top`, `.lightbox-close`
3. **Print styles**: Hide navigation, back-to-top, comments, and other non-content elements; show URLs after links

### Lightbox Focus Trap Hardening

The original focus trap only checked `activeElement === first/last`. If focus escaped to an element outside the focusable list (e.g., shadow DOM or browser extension injections), the trap would fail. Now also checks whether `activeElement` is within the focusable list.

### Blog List Page SEO

`blog/page.tsx`'s `generateMetadata` only set `title`, missing `description` and `openGraph`. Added a `blog.description` i18n key and completed the metadata.

### TableOfContents Non-null Assertion Fix

Line 73 used `mutationObserver?.observe(article!, ...)` — a non-null assertion after optional chaining. If `article` is null, this crashes. Changed to an explicit null check.

## Round 14: Deep Review and Robustness Hardening

Another comprehensive code review covering security, accessibility, robustness, and SEO.

### Mobile Navigation Keyboard Support

The mobile hamburger menu had no keyboard close mechanism, and screen reader users weren't informed when the menu opened. Two improvements:

1. Press `Escape` to close the menu
2. Auto-focus the first link when the menu opens

### RSS Feed Enhancements

- Added `<category>` tags for each post's tags, enabling RSS readers to categorize content
- Added `isNaN` check to prevent invalid dates from corrupting `lastBuildDate`

### OG Image Route Locale Validation

The `/og` route's `locale` parameter previously accepted any string. Now validates against `routing.locales`, falling back to `"zh"` for invalid values.

### Environment URL Validation

`getSiteUrl()` now validates URL format with `new URL()` and strips trailing slashes, preventing double-slash URLs like `https://example.com//zh/blog/...`.

### Dark Mode Script Robustness

The inline script's `catch` block was empty — in private browsing mode where `localStorage` is unavailable, users would see a theme flash. The `catch` block now falls back to the `prefers-color-scheme` media query.

### Security Header Addition

Added `X-XSS-Protection: 0` — the modern best practice is to disable the browser's built-in XSS filter (relying on CSP instead).

### Other Small Fixes

- Added `type="button"` to `error.tsx` retry button to prevent form submission in form contexts
- Removed unnecessary `rel="noopener noreferrer"` from `mailto` links in Footer
- Added `role="search"` to SearchBar form and `autoComplete="off"` to the input
- Wrapped photos page `fs.readdirSync` in `try-catch` to handle permission errors gracefully

## Summary

14 rounds, 100+ improvements, 20+ files touched. Key wins:

- **Security**: Response headers, disabled X-Powered-By, OG parameter validation
- **Accessibility**: Lightbox focus trap, scroll lock, aria attributes, focus ring visibility, UI label i18n, `prefers-reduced-motion`, `focus-visible` rings, print styles
- **Performance**: MutationObserver over setTimeout, Giscus postMessage, conditional video loading, TOC caching
- **Consistency**: 18 CSS design tokens, full i18n coverage, centralized env handling, bilingual data files
- **Maintainability**: useDarkMode hook, data file extraction, type safety, error logging, redundant code cleanup
- **SEO**: canonical/alternate URLs, structured data, RSS normalization

A code review isn't just about finding problems — it's about establishing conventions. Once a project has unified design tokens, i18n coverage, and error handling patterns, future development naturally follows those standards.
