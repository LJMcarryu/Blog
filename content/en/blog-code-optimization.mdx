## Round 1: High-Priority Fixes

### Centralized Environment Variables

Five files each had their own copy of `process.env.NEXT_PUBLIC_SITE_URL || "https://example.com"` — `robots.ts`, `sitemap.ts`, `feed.xml/route.ts`, `layout.tsx`, and `blog/[slug]/page.tsx`.

Created `lib/env.ts` with a `getSiteUrl()` function that handles the fallback and warns when the env var is missing:

```typescript
export function getSiteUrl(): string {
  const url = process.env.NEXT_PUBLIC_SITE_URL;
  if (!url) {
    console.warn("[env] NEXT_PUBLIC_SITE_URL is not set — falling back to", FALLBACK_URL);
    return FALLBACK_URL;
  }
  return url;
}
```

### Lightbox Accessibility

The original Lightbox was a `div` pretending to be a modal — keyboard users could Tab right out of it. After the fix:

- Added `aria-modal="true"`
- Auto-focus the close button on open
- Lock body scroll
- Implemented a focus trap (Tab cycles within the lightbox)

### CSS Cleanup

- Code block backgrounds `#fafafa` / `#161b22` → `var(--code-bg)`
- Code text colors `#0a0a0a` / `#f0f0f0` → `var(--fg-deeper)` / `var(--fg)`
- Added comments explaining why each `!important` is necessary

### Other Fixes

- OG image title truncated to 100 characters
- MDX parse errors now log with `console.warn`
- Hardcoded "All" / "Previous" / "Next" labels moved to i18n messages
- RSS `language` corrected from `"zh-en"` to `"zh"`
- Layout metadata description changed to SEO-friendly English
- `CodeCopyButton` replaced `setTimeout` with `MutationObserver`
- `ThemedVideo` switched to dual video elements with CSS visibility toggle
- SKILLS data extracted from about page to `data/skills.ts`
- i18n type assertion now uses `(typeof routing.locales)[number]`

## Round 2: Component Optimization

### TableOfContents Timing Fix

Same pattern as CodeCopyButton — replaced the 150ms `setTimeout` with a `MutationObserver` to reliably detect when MDX content finishes rendering.

### Pagination i18n

The "Previous" / "Next" labels in `Pagination.tsx` were also hardcoded. Changed to accept translated labels via props, removing the `locale` prop entirely.

### Giscus Theme Switch Optimization

Previously, every theme toggle destroyed and recreated the entire Giscus iframe. Now uses the Giscus `postMessage` API to update the theme in-place:

```typescript
iframe.contentWindow?.postMessage(
  { giscus: { setConfig: { theme } } },
  "https://giscus.app"
);
```

### Minor Fixes

- Star rating color `#facc15` → `var(--star)` CSS variable
- ThemedVideo only sets `autoPlay` and `preload="auto"` on the visible video

```typescript
const nextConfig: NextConfig = {
  poweredByHeader: false,

        { key: "X-Content-Type-Options", value: "nosniff" },
```

## Round 5: Design Token System

The biggest CSS refactor. The project used `rgba(125, 125, 125, 0.2)` and `rgba(125, 125, 125, 0.1)` extensively for borders and backgrounds, scattered across CSS and JSX inline styles.

Added two new design tokens:

```css
:root {
  --border:     rgba(125, 125, 125, 0.2);
  --bg-subtle:  rgba(125, 125, 125, 0.1);
}
```

Replaced 9 CSS selectors and 7 JSX inline styles to use the variables consistently.

Also fixed an unnecessary `as unknown as` double type cast in `ThemeToggle.tsx` — the project's TypeScript DOM lib already supports `startViewTransition`.

## Round 6: Deep Cleanup and Accessibility

The second review uncovered issues in three areas: incomplete CSS variable coverage, accessibility gaps, and redundant code.

### CSS Variable System Completion

`globals.css` still had 15+ raw `rgba(125, 125, 125, ...)` values. Added 7 new design tokens to eliminate every last one:

```css
:root {
  --border-light:    rgba(125, 125, 125, 0.15);
  --border-strong:   rgba(125, 125, 125, 0.25);
  --border-faint:    rgba(125, 125, 125, 0.3);
  --scrollbar:       rgba(125, 125, 125, 0.35);
  --scrollbar-hover: rgba(125, 125, 125, 0.6);
  --code-label-bg:   rgba(125, 125, 125, 0.08);
  --inline-code-bg:  rgba(125, 125, 125, 0.12);
}
```

Replaced values across: scrollbar, code block borders, language label backgrounds, inline code, blockquotes, TOC sidebar, mobile menu border, and JSX inline styles (photos page empty state, post navigation divider).

### Giscus Initialization Fix

Changed the init effect dependency from `[locale, theme]` to `[locale]`, and read the actual DOM theme state at initialization time:

```typescript
const isDark = document.documentElement.classList.contains("dark");
const giscusTheme = isDark ? "dark" : "light";
```

Now the Giscus iframe loads with the correct theme on first render — no more flash.

### Accessibility Improvements

- Hamburger button gets `aria-controls="mobile-menu"`, menu gets matching `id`
- Active tag filter gets `aria-current="true"`
- Lightbox now passes through the original image's `alt` text instead of hardcoded `"Enlarged preview"`
- Search input gets `focus:ring-current` for visible focus ring

### Redundant Code Cleanup

- Removed redundant `dynamic = 'force-static'` from `blog/[slug]/page.tsx`
- Removed unnecessary `target="_self"` from mailto links
- RSS feed description changed from mixed-language to Chinese only

## Round 7: Missed Spots and Cleanup

A third deep review caught several issues that slipped through earlier rounds.

### SearchBar's Missed Raw rgba

The design token migration missed `rgba(125, 125, 125, 0.05)` in `SearchBar.tsx`. Added a new `--search-bg` token to cover it.

### Pagination Link Trailing `?`

`blog/page.tsx` returned `"?"` as `baseHref` when no filter params were active, producing URLs like `/blog?`. Fixed to return an empty string — the `Pagination` component already handles `?page=2` concatenation correctly.

### Giscus Language Switch Broken

`Comments.tsx` used an `initialized` ref to prevent duplicate initialization, but this also blocked re-initialization when the locale changed. Removed the guard and switched to clearing the container on locale change:

```typescript
useEffect(() => {
  const container = ref.current;
  if (!container) return;
  container.innerHTML = "";
  // ... re-create script
}, [locale]);
```

### CSS Variable Deduplication

The `.dark` block contained 11 CSS variables identical to `:root` (all rgba-based tokens plus `--star` and `--success`). These values don't change between modes — the duplication was pure noise. After cleanup, `.dark` only contains the 6 color variables that actually differ.

### RSS Description Language Fix

Changed the RSS `<description>` from Chinese-only to English (`"Jimmy's personal blog — mobile development, tech, and more"`), since the feed includes posts in both languages.

## Summary

7 rounds, 45+ improvements, 20+ files touched. Key wins:

- **Security**: Response headers, disabled X-Powered-By, OG parameter validation
- **Accessibility**: Lightbox focus trap, scroll lock, aria attributes, focus ring visibility
- **Performance**: MutationObserver over setTimeout, Giscus postMessage, conditional video loading
- **Consistency**: 18 CSS design tokens, full i18n coverage, centralized env handling
- **Maintainability**: Data file extraction, type safety, error logging, redundant code cleanup

A code review isn't just about finding problems — it's about establishing conventions. Once a project has unified design tokens, i18n coverage, and error handling patterns, future development naturally follows those standards.
