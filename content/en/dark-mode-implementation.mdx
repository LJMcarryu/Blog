## Dark Mode Is More Than a Background Swap

Many websites have a common dark mode problem: **a flash of white on refresh**. The user chose dark theme, yet every page load briefly shows the light theme before switching. Terrible UX.

Today I'm sharing a complete dark mode solution that addresses three core problems:

1. **Zero flash**: No white flash on refresh
2. **System preference**: Automatically follow OS setting
3. **Smooth transitions**: Elegant animations via View Transitions API

## The Architecture

### CSS Variable System

First, define a set of design tokens:

```css
:root {
  --c-bg:     #ffffff;
  --fg:       #555555;
  --fg-deep:  #222222;
  --fg-deeper:#000000;
  --fg-light: #888888;
}

.dark {
  --c-bg:     #050505;
  --fg:       #bbbbbb;
  --fg-deep:  #dddddd;
  --fg-deeper:#ffffff;
  --fg-light: #888888;
}

html {
  background-color: var(--c-bg);
  color-scheme: light;
}

html.dark {
  color-scheme: dark;
}
```

CSS variables over Tailwind's `dark:` prefix because variables work everywhere — including third-party library style overrides.

### Zero Flash: Inline Blocking Script

The key insight: **determine the theme before the browser paints the first frame**.

```tsx
// layout.tsx — placed in <head>
<script
  dangerouslySetInnerHTML={{
    __html: `(function(){
      try {
        var d = localStorage.getItem('blog-color-scheme') || 'auto';
        var sys = window.matchMedia('(prefers-color-scheme:dark)').matches;
        if (d === 'dark' || (d === 'auto' && sys))
          document.documentElement.classList.add('dark');
      } catch(e) {}
    })()`,
  }}
/>
```

This script is **synchronous and blocking** — the browser executes it immediately upon parsing, before rendering anything. Users never see the flash.

### React State Synchronization

```tsx
"use client";
import { useEffect, useState } from "react";

export default function ThemeToggle() {
  const [dark, setDark] = useState(false);

  useEffect(() => {
    function applyTheme() {
      try {
        const saved = localStorage.getItem("blog-color-scheme");
        const prefersDark = window.matchMedia(
          "(prefers-color-scheme:dark)"
        ).matches;
        const shouldBeDark =
          saved === "dark" || (saved !== "light" && prefersDark);

        document.documentElement.classList.toggle("dark", shouldBeDark);
        setDark(shouldBeDark);
      } catch {
        setDark(document.documentElement.classList.contains("dark"));
      }
    }

    applyTheme();

    // MutationObserver: prevent React hydration from removing dark class
    const observer = new MutationObserver(applyTheme);
    observer.observe(document.documentElement, {
      attributes: true,
      attributeFilter: ["class"],
    });

    return () => observer.disconnect();
  }, []);

  // ...
}
```

`MutationObserver` is the core defense mechanism. When React hydrates the server-rendered HTML, it may reset the `<html>` class attribute. The observer fires as a **microtask** — before the browser's next paint — so users never see any flash.

### View Transitions API: Smooth Animation

```typescript
const toggle = () => {
  const html = document.documentElement;
  const next = !html.classList.contains("dark");

  const apply = () => {
    html.classList.toggle("dark", next);
    setDark(next);
    try {
      localStorage.setItem("blog-color-scheme", next ? "dark" : "light");
    } catch {}
  };

  // Use View Transitions for smooth animation if available
  if ("startViewTransition" in document) {
    (document as unknown as {
      startViewTransition: (cb: () => void) => void;
    }).startViewTransition(apply);
  } else {
    apply();
  }
};
```

The View Transitions API automatically captures before/after snapshots and creates a cross-fade animation. The result is buttery smooth.

Pair it with CSS to control the animation duration:

```css
::view-transition-old(root),
::view-transition-new(root) {
  animation-duration: 0.35s;
}
```

## The Locale-Switch Trap

With `next-intl` locale-based routing, switching languages remounts the entire component tree. This means React may lose the `dark` class when rendering the new locale page.

The solution is the `MutationObserver` mentioned above — it detects any class change and immediately restores the correct theme.

```tsx
// Timeline:
// 1. User switches language zh → en
// 2. React unmounts old components, mounts new ones
// 3. New <html> may lose the dark class
// 4. MutationObserver microtask fires immediately
// 5. applyTheme() re-adds the dark class
// 6. Browser paints — user always sees dark mode
```

## Complete Flow

```
User visits page
  ↓
<head> blocking script executes
  ↓
Read localStorage → determine theme → add .dark class
  ↓
Browser first paint (already correct theme)
  ↓
React hydrates → MutationObserver is active
  ↓
User clicks toggle → View Transition animation → localStorage persists
```
