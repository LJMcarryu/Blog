## 工作原理

本项目使用 [next-intl](https://next-intl.dev) 实现国际化，采用**基于 URL 的语言路由**：

```
/zh/blog     → 中文博客列表
/en/blog     → 英文博客列表
```

### 请求处理流程

```
用户访问 /about
    ↓
proxy.ts 中间件
    ↓ 检测到没有语言前缀
重定向到 /zh/about（defaultLocale = "zh"）
    ↓
app/[locale]/about/page.tsx 渲染
    ↓ locale = "zh"
getTranslations({ locale: "zh", namespace: "about" })
    ↓ 从 messages/zh.json 读取 about 命名空间
返回翻译后的页面
```

### 语言切换

用户点击右上角的语言切换按钮时：

```typescript
// components/LanguageSwitcher.tsx
router.replace(pathname, { locale: nextLocale });
// 例：当前在 /zh/blog，切换后跳转到 /en/blog
// pathname 保持不变，只有语言前缀改变
```

## 目录结构

```
i18n/
├── routing.ts      # 定义支持的语言和默认语言
├── navigation.ts   # 导出 locale-aware 的路由工具
└── request.ts      # 服务端：每次请求时加载对应语言的翻译

messages/
├── zh.json         # 中文 UI 字符串
└── en.json         # 英文 UI 字符串

proxy.ts            # next-intl 中间件
```

### `i18n/routing.ts`

```typescript
import { defineRouting } from "next-intl/routing";

export const routing = defineRouting({
  locales: ["zh", "en"],    // 支持的语言列表
  defaultLocale: "zh",      // 默认语言（无前缀时跳转到此语言）
});
```

### `i18n/navigation.ts`

```typescript
import { createNavigation } from "next-intl/navigation";
import { routing } from "./routing";

// 导出 locale-aware 的路由组件和 hooks
export const { Link, redirect, usePathname, useRouter } =
  createNavigation(routing);
```

使用这里导出的 `Link` 而非 `next/link` 的 `Link`，可以自动在链接中注入当前语言前缀。

## 翻译字符串管理

### 文件结构（以 `messages/zh.json` 为例）

```json
{
  "nav": {               // namespace: "nav"
    "home": "首页",
    "blog": "博客"
  },
  "home": {              // namespace: "home"
    "name": "刘佳明",
    "tagline": "..."
  },
  "blog": {              // namespace: "blog"
    "title": "所有文章",
    "noResults": "没有找到相关文章"
  },
  "about": {             // namespace: "about"
    "title": "关于我",
    "bio1": "..."
  }
  // 更多 namespace...
}
```

### 命名规范

- **Namespace** 对应页面名称（小写，如 `blog`、`about`、`projects`）
- **Key** 使用小驼峰命名（如 `backToList`、`noResults`）
- 中英文文件结构**完全一致**，只是值不同

## 在服务端组件中使用翻译

```typescript
// app/[locale]/example/page.tsx
import { getTranslations } from "next-intl/server";

export default async function ExamplePage({
  params,
}: {
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: "example" });

  return (
    <div>
      <h1>{t("title")}</h1>        {/* messages/zh.json → example.title */}
      <p>{t("description")}</p>
    </div>
  );
}
```

**注意：** `getTranslations` 是异步函数，需要 `await`，只能在 `async` 函数组件中使用。

## 在客户端组件中使用翻译

```typescript
// components/Navigation.tsx
"use client";

import { useTranslations } from "next-intl";

export default function Navigation() {
  const t = useTranslations("nav");  // 同步，无需 await

  return <nav>{t("home")}</nav>;
}
```

**注意：** 客户端组件的翻译在服务端渲染时由 `NextIntlClientProvider`（在 layout.tsx 中）预先注入到页面，所以客户端不需要额外请求。

## 语言感知的导航

使用 `i18n/navigation.ts` 导出的 `Link`，链接会自动带上当前语言前缀：

```typescript
import { Link } from "@/i18n/navigation";

// 当前语言为 zh 时：
<Link href="/blog">博客</Link>
// 渲染为: <a href="/zh/blog">博客</a>

// 当前语言为 en 时：
<Link href="/blog">Blog</Link>
// 渲染为: <a href="/en/blog">Blog</a>
```

**不要使用** `next/link` 的 `Link`，否则跳转时会丢失语言前缀。

## 新增翻译 Namespace

以添加 `gallery` 页面的翻译为例：

### 第一步：在 `messages/zh.json` 添加

```json
{
  "gallery": {
    "title": "图库",
    "subtitle": "我的作品集"
  }
}
```

### 第二步：在 `messages/en.json` 添加对应翻译

```json
{
  "gallery": {
    "title": "Gallery",
    "subtitle": "My portfolio"
  }
}
```

**重要：** 两个文件的 namespace 名称和 key 必须完全相同，只有值不同。

## 新增页面的完整 i18n 流程

以新建一个 "Gallery" 页面为例，完整流程：

### 1. 添加翻译字符串

`messages/zh.json`：
```json
{
  "nav": {
    "gallery": "图库"
  },
  "gallery": {
    "title": "图库",
    "subtitle": "我的摄影与设计作品"
  }
}
```

`messages/en.json`：
```json
{
  "nav": {
    "gallery": "Gallery"
  },
  "gallery": {
    "title": "Gallery",
    "subtitle": "My photography and design work"
  }
}
```

### 2. 创建页面文件

`app/[locale]/gallery/page.tsx`：
```typescript
import { getTranslations } from "next-intl/server";

export default async function GalleryPage({
  params,
}: {
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: "gallery" });

  return (
    <div>
      <h1>{t("title")}</h1>
      <p>{t("subtitle")}</p>
    </div>
  );
}
```

### 3. 添加导航链接

`components/Navigation.tsx`：
```typescript
const navLinks = [
  // ...现有链接
  { href: "/gallery", label: t("gallery") },
];
```

### 4. 验证

访问 `/zh/gallery` 和 `/en/gallery` 确认两种语言都正常显示。

## 语言相关的条件渲染

有时需要在代码中根据语言做不同处理：

```typescript
// 方式一：通过 locale 参数判断（推荐用于服务端组件）
const content = locale === "zh" ? ZH_CONTENT : EN_CONTENT;

// 方式二：直接访问 locale
import { useLocale } from "next-intl"; // 客户端组件
const locale = useLocale();
```

**尽量避免在代码中硬编码语言字符串**，优先使用 `messages/*.json` 文件统一管理翻译。只有当内容本身（如书单、工具列表）需要维护两份独立数据时，才在页面文件中用 `ZH_xxx`/`EN_xxx` 分开定义。
